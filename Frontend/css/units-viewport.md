# Динамические единицы измерения Viewport (vw, vh, dvh, svh, lvh, vmin, vmax)

## Что я узнал

Viewport Units — это единицы измерения CSS, которые зависят от **размеров области просмотра браузера (viewport)**. Они позволяют создавать **адаптивные интерфейсы без медиазапросов**, потому что размеры элементов автоматически пересчитываются относительно ширины или высоты экрана.

Базовые единицы viewport:

- `vw` — 1% ширины viewport
- `vh` — 1% высоты viewport

Например:

- `100vw` = вся ширина экрана
- `100vh` = вся высота экрана

Однако на **мобильных устройствах** классическая единица `vh` часто работает нестабильно. Причина — **динамическая адресная строка браузера**.

Когда пользователь прокручивает страницу:

- адресная строка может **появляться**
- или **скрываться**

Из-за этого высота viewport изменяется, и элементы с `100vh` могут **прыгать или обрезаться**.

Чтобы решить эту проблему, появились **новые viewport-единицы**.

---

### dvh — Dynamic Viewport Height

`dvh` означает **Dynamic Viewport Height**.

Эта единица всегда учитывает **реальную текущую высоту экрана**, включая:

- появление адресной строки
- скрытие адресной строки
- системные панели

Это делает `dvh` **самым точным способом задать высоту экрана на мобильных устройствах**.

Например:

```

height: 100dvh;

```

Элемент всегда будет занимать **текущую доступную высоту**.

---

### svh и lvh

Кроме динамического значения появились ещё две единицы.

**svh — Small Viewport Height**

Это **минимально возможная высота viewport**, когда все интерфейсы браузера видны.

То есть:

- адресная строка
- панели браузера

включены.

---

**lvh — Large Viewport Height**

Это **максимально возможная высота viewport**, когда браузерные панели скрыты.

Используется для случаев, когда нужно учитывать **максимальное пространство экрана**.

---

### vmin и vmax

Эти единицы выбирают **минимальное или максимальное значение между шириной и высотой viewport**.

`vmin`:

```

1vmin = 1% меньшей стороны экрана

```

`vmax`:

```

1vmax = 1% большей стороны экрана

```

Это удобно для:

- адаптивной типографики
- квадратных элементов
- масштабируемых интерфейсов

Например:

```

font-size: 5vmin;

```

Текст будет масштабироваться относительно **меньшей стороны экрана**, что делает его более универсальным.

---

### Когда использовать разные единицы

`vh`  
подходит для простых случаев и десктопных сайтов.

`dvh`  
лучший вариант для **мобильных интерфейсов и полноэкранных блоков**.

`svh`  
используется, если нужно учитывать **минимальную высоту viewport**.

`lvh`  
полезен, если интерфейс должен использовать **максимально возможную высоту экрана**.

`vmin` и `vmax`  
подходят для **адаптивной графики и типографики**.

---

### Пример
```javascript
/* Полноэкранный баннер */
.banner {
  height: 100dvh;
  display: grid;
  place-items: center;
  background-color: lightskyblue;
}

/* Минимальная высота viewport */
.section {
  min-height: 100svh;
}

/* Максимальная высота viewport */
.fullscreen-gallery {
  height: 100lvh;
}

/* Адаптивный текст */
.title {
  font-size: 6vmin;
}

/* Квадратный элемент */
.box {
  width: 40vmin;
  height: 40vmin;
  background-color: lightcoral;
}

/* Использование vmax */
.hero-title {
  font-size: 8vmax;
}
```
