# Устаревшие CSS-свойства, вендорные префиксы и поддержка браузеров

## Что я узнал

В CSS существуют **устаревшие свойства**, **вендорные префиксы** и механизмы **проверки поддержки возможностей браузера**. Понимание этих вещей важно для обеспечения корректной работы сайта в разных браузерах.

### Устаревшие свойства

Со временем некоторые CSS-свойства становятся устаревшими и заменяются новыми, более универсальными вариантами. Обычно это происходит потому, что спецификация меняется или появляется более удобный стандарт.

Примеры таких замен:

- `grid-column-gap` → `column-gap`
- `grid-row-gap` → `row-gap`
- `offset-*` → `inset-*`

Новые свойства работают в большем количестве контекстов и лучше поддерживаются современными браузерами.

Также встречаются **полностью устаревшие свойства**, которые больше не используются в современных проектах, например:

- `ime-mode`
- `pause-before`
- `rest-before`
- `cue-before`

Такие свойства остались в старых спецификациях и практически не поддерживаются современными браузерами.

### Вендорные префиксы

Раньше браузеры внедряли новые CSS-возможности как **экспериментальные**, поэтому они требовали специального префикса производителя браузера.

Наиболее распространённые префиксы:

- `-webkit-` — Chrome, Safari
- `-moz-` — Firefox
- `-ms-` — Internet Explorer
- `-o-` — Opera (старые версии)

Префиксы могли применяться к:

1. **Свойствам**
2. **Значениям**
3. **Псевдоклассам**
4. **директивам (`@keyframes`)**

Сегодня большинство современных браузеров поддерживают стандартные версии свойств, но префиксы могут использоваться для обратной совместимости.

### Проверка поддержки CSS

CSS предоставляет специальную директиву `@supports`, которая позволяет применять стили **только если браузер поддерживает определённое свойство**.

Это помогает безопасно внедрять новые технологии без поломки интерфейса в старых браузерах.

### Фоллбэк (Fallback)

Фоллбэк — это **запасное значение**, которое используется, если браузер не поддерживает более современную функцию.

Например, сначала указывается простой фон, а затем более сложный вариант.

### Инструменты для проверки поддержки

Существует несколько полезных инструментов для работы с совместимостью браузеров:

- **Can I Use** — показывает поддержку CSS-функций
- **Autoprefixer** — автоматически добавляет нужные вендорные префиксы
- **CSS Preset Env** — позволяет использовать будущие CSS-возможности уже сейчас

### Пример
```javascript
.box, .box1 {
  width: 100px;
  height: 100px;
  background-color: lightskyblue;

  /* Устаревшее */
  grid-column-gap: 30px;

  /* Современное */
  column-gap: 30px;

  /* Устаревшее */
  grid-row-gap: 30px;

  /* Современное */
  row-gap: 30px;

  /* Новые логические свойства */
  inset-block-end: 5px;
  inset-block-start: 5px;
  inset-inline-end: 5px;
  inset-inline-start: 5px;
}

.box1 {
  background-color: salmon;

  /* Вендорные префиксы для свойства */
  -webkit-transition: 0.2s ease;
  -moz-transition: 0.2s ease;
  -ms-transition: 0.2s ease;
  -o-transition: 0.2s ease;
  transition: 0.2s ease;

  /* Вендорные префиксы для значения display */
  display: -webkit-box;
  display: -webkit-flex;
  display: -ms-flexbox;
  display: flex;
}

/* Вендорные префиксы для placeholder */
input::-webkit-input-placeholder { color: brown }
input::-moz-placeholder { color: red }
input::-ms-input-placeholder { color: blueviolet }
input::placeholder { color: blue }

/* Префиксы для keyframes */
@-webkit-keyframes some-animation {
  from { opacity: 0 }
  to { opacity: 1 }
}

@keyframes some-animation {
  from { opacity: 0 }
  to { opacity: 1 }
}

/* Экспериментальные свойства */
html {
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}

/* Фоллбэк */
body {
  background: url('...') center/cover no-repeat #eeeeee;
}

/* Проверка поддержки */
@supports (mask-clip: view-box) {
  body {
    mask-clip: view-box;
  }
}
```
