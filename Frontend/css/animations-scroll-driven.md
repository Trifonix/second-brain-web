# Scroll-driven анимации в CSS

## Что я узнал

Современный CSS начал поддерживать **scroll-driven animations** — анимации, которые запускаются и управляются **не временем**, а **прокруткой страницы**.

В классической модели CSS-анимации используется `@keyframes` и свойство `animation-duration`. Анимация проигрывается **по таймеру**.

Однако новые возможности CSS позволяют привязать анимацию к **позиции скролла**. Это означает, что:

- анимация двигается **вместе с прокруткой страницы**
- прогресс анимации зависит от **позиции пользователя на странице**
- можно создавать **интерактивные эффекты без JavaScript**

Эта технология называется **Scroll-driven animations**.

Главные инструменты:

- `scroll-timeline`
- `animation-timeline`
- `scroll()`
- `view-timeline`
- `animation-range`

Эти свойства пока считаются **экспериментальными**, но уже поддерживаются в современных браузерах (например, Chromium-based).

---

### Scroll Timeline

**Scroll Timeline** позволяет привязать анимацию к **скроллу контейнера**.

Вместо времени (`animation-duration`) используется **позиция прокрутки**.

Когда пользователь прокручивает страницу:

- анимация постепенно проигрывается
- если прокрутить назад — анимация тоже откатится

Это идеально подходит для:

- параллакса
- прогресс-индикаторов
- интерактивных лендингов
- анимированных историй (scrollytelling)

Пример логики:

1. создается `scroll-timeline`
2. элементу задается `animation-timeline`
3. анимация синхронизируется со скроллом

---

### View Timeline

**View Timeline** — ещё более мощный инструмент.

Он запускает анимацию **когда элемент появляется в viewport** (области видимости).

Это похоже на **Intersection Observer**, но полностью на CSS.

Можно управлять моментом запуска анимации через:

```

animation-range

```

Это позволяет задать:

- когда анимация начинается
- когда она заканчивается
- какой участок скролла используется

Например:

- анимация может начаться, когда элемент **входит в экран**
- закончиться, когда он **полностью прокручен**

---

### Преимущества scroll-driven animations

1. **Не нужен JavaScript**
2. **Высокая производительность**
3. Анимации синхронизированы со скроллом
4. Подходит для сложных интерфейсов и storytelling
5. Более плавная работа на мобильных устройствах

---

### Когда применять

Scroll-driven animations особенно полезны для:

- лендингов
- интерактивных презентаций
- эффектов появления элементов
- анимированных графиков
- визуальных историй

Но важно помнить:

- поддержка ещё развивается
- иногда требуется fallback

---

### Пример
```javascript
@keyframes fade-slide {
  from {
    opacity: 0;
    translate: 0 50px;
  }
  to {
    opacity: 1;
    translate: 0 0;
  }
}

/* Scroll timeline */
.scroller {
  height: 200vh;
  scroll-timeline-name: --pageScroll;
  scroll-timeline-axis: block;
}

.box {
  width: 150px;
  height: 150px;
  background-color: lightskyblue;

  animation-name: fade-slide;
  animation-duration: 1s;

  /* привязка анимации к прокрутке */
  animation-timeline: --pageScroll;
}

/* View timeline */

.card {
  view-timeline-name: --cardReveal;
  view-timeline-axis: block;
}

.card-content {
  animation-name: fade-slide;
  animation-duration: 1s;

  animation-timeline: --cardReveal;

  /* запуск анимации при появлении элемента */
  animation-range: entry 0% cover 40%;
}
```
