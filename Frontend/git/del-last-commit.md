# How to delete last commit from GitHub branch

## Что я узнал

Иногда в репозитории на GitHub нужно удалить последний коммит. Это может быть полезно, если был сделан неправильный коммит или вы случайно запушили лишние изменения. Основной подход — откат локальной ветки на один коммит назад и принудительная отправка на удалённый репозиторий.

1. **Откат локальной ветки на один коммит**
   ```bash
   git reset --hard HEAD~1
```

Команда `HEAD~1` указывает на предыдущий коммит. Опция `--hard` удаляет изменения из индекса и рабочего каталога, полностью возвращая состояние репозитория к предыдущему коммиту.

2. **Принудительная отправка изменений на GitHub**

   ```bash
   git push --force origin master
   ```

   Опция `--force` переписывает ветку на удалённом репозитории, удаляя последний коммит на сервере.

> Важно: использовать `--force` осторожно, особенно если над веткой работают несколько человек, чтобы не потерять чужие изменения.

3. **Уточнение названия ветки**
   В зависимости от репозитория основной веткой может быть `main` или `master`. Команды будут идентичными, просто меняется имя ветки:

   ```bash
   git push --force origin main
   ```

### Пример

```bash
# Удаляем последний коммит и синхронизируем с GitHub
git reset --hard HEAD~1
git push --force origin main
```
