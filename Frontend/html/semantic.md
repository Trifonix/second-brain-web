# Семантический HTML

## Что я узнал

Семантический HTML — это использование тегов по их смысловому назначению, а не просто для визуального отображения.

Главная идея — структура страницы должна быть понятна:
- браузеру
- поисковым системам
- скринридерам
- другим разработчикам

Семантические теги описывают роль элемента на странице.  
Например, `<header>` — это шапка сайта или секции, `<main>` — основной контент, `<footer>` — подвал.

Это улучшает:
- SEO
- доступность (a11y)
- читаемость кода
- архитектуру проекта

### Основные семантические теги

- `<header>` — верхняя часть страницы или секции
- `<nav>` — навигация
- `<main>` — основной контент (только один на странице)
- `<section>` — логический раздел
- `<article>` — независимый контент (пост, статья, карточка)
- `<aside>` — боковая информация
- `<footer>` — нижняя часть страницы или секции

### Когда использовать article, а когда section?

`article` — если контент можно вынести отдельно (например, пост блога).  
`section` — если это просто логический блок внутри страницы.

### Почему нельзя всё делать через div?

`div` не несёт смысла.  
Если всё сверстано через `div`, структура страницы становится нечитаемой для поисковых систем и технологий доступности.

Правило:  
Сначала думаем о семантике → потом о стилях.

---

### Пример
```javascript
<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <title>Пример семантической структуры</title>
</head>
<body>
  <header>
    <h1>Мой блог</h1>
    <nav>
      <a href="#">Главная</a>
      <a href="#">Статьи</a>
      <a href="#">Контакты</a>
    </nav>
  </header>

  <main>
    <article>
      <h2>Заголовок статьи</h2>
      <p>Текст статьи...</p>
    </article>

    <aside>
      <p>Популярные статьи</p>
    </aside>
  </main>

  <footer>
    <p>© 2026 Мой блог</p>
  </footer>
</body>
</html>
```
